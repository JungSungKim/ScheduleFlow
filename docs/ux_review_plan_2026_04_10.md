# ScheduleFlow UX 개선 통합 기획안 (v2, 2026.04.10)

리뷰 피드백 5건을 분석하고, UI 용어를 먼저 정의한 뒤 구현 방향을 정의합니다.

---

## 0. ScheduleFlow UI 용어 사전

앱 내 요소에 공통 명칭을 부여하여 이후 소통에서 혼동이 없도록 정의합니다.

### 화면(Screen) 구분

| 명칭 | 진입 방법 | 설명 |
|------|-----------|------|
| **대시보드** | 사이드바 > "대시보드" 또는 홈(🏠) 버튼 | 전체 현황을 한눈에 보는 요약 화면 |
| **TODO 페이지** | 사이드바 > "TODO" | 할 일 전체 목록 관리 화면 |
| **캘린더 페이지** | 사이드바 > "캘린더" | 월간 일정 캘린더 전체 화면 |
| **출장 페이지** | 사이드바 > "출장" | 출장 목록 전체 관리 화면 |
| **문서 페이지** | 사이드바 > "문서" | 출장 관련 문서(사전신청서/보고서) 화면 |

### 대시보드 내 구성요소

| 명칭 | 위치 | 설명 |
|------|------|------|
| **요약 카드** | 대시보드 상단 그리드 | "오늘의 할 일", "다가오는 출장", "미완료 문서" 각 카드 |
| **미니 캘린더** | 대시보드 하단 | 대시보드 안의 축소된 월간 캘린더 |
| **할 일 항목** (in 요약 카드) | "오늘의 할 일" 카드 내부 | 대시보드에서 보이는 TODO 한 줄 |
| **출장 항목** (in 요약 카드) | "다가오는 출장" 카드 내부 | 대시보드에서 보이는 출장 한 줄 |

### 공통 UI 컴포넌트

| 명칭 | 설명 |
|------|------|
| **상세 패널** | 항목을 클릭했을 때 오른쪽이나 하단에서 슬라이드로 나타나는 읽기 전용 상세 보기 |
| **수정 폼** (Form/Modal) | 새로 만들거나 수정할 때 열리는 모달 입력창 |
| **상태 배지** | 할 일/출장의 현재 상태를 표시하는 색깔 라벨 (예: 계획됨, 진행중, 완료) |
| **D-Day 배지** | 출장 카드 오른쪽의 남은 날수 표시 (D-1, D-Day, D+1) |
| **일정 바 (Event Bar)** | 캘린더에서 날짜를 연결하는 가로 막대 형태의 이벤트 표현 |

> [!NOTE]
> 앞으로 소통할 때 위 명칭을 사용하면 설명 없이도 정확하게 전달됩니다.  
> 예: "**대시보드 > 출장 항목**을 클릭하면 **상세 패널**이 열리고, 거기서 **수정 폼**으로 넘어가게 해줘"

## 리뷰 항목 1 — 캘린더 일정 바 색상 시인성 개선

### 현황 분석
캘린더 페이지에서 오늘 날짜 셀(`.today`)이 `--color-primary`(주황) 배경을 가집니다.  
TODO 타입 **일정 바**도 동일한 `--color-primary` 계열 배경을 사용하여  
**오늘 날짜 셀 위의 TODO 일정 바가 배경에 묻혀 시인성이 크게 저하됩니다.**

### 해결 방향
- `.cal-day.today` 내부의 `.cal-event.todo` 전용 스타일 추가 → 흰색 반투명 반전 처리
- 영향 파일: `index.css` 만 (JS 수정 없음)

## 리뷰 항목 2 — 캘린더 일정 바 호버 피드백

### 현황 분석
캘린더 페이지의 **일정 바**에 onclick은 붙어 있으나  
마우스를 올렸을 때 클릭 가능함을 알려주는 **시각적 피드백(Hover 강조)**이 없습니다.

### 해결 방향
- `.cal-event` CSS에 `cursor: pointer` 및 hover 시 `brightness` + 미세 `scaleY` 애니메이션 추가
- 영향 파일: `index.css`

## 리뷰 항목 3 — TODO ↔ 출장 연계 워크플로우

### 현황 분석
출장 관련 할 일을 등록하려면 **출장 페이지 등록 → TODO 페이지 별도 등록** 2번 작업이 필요합니다.  
두 데이터 사이에 연결 고리가 없어 맥락이 단절됩니다.

### 기획: `tripId` 참조 필드 선택적 도입

#### 데이터 모델 변경 (하위 호환)
```javascript
// TODO 데이터 모델 확장 (선택 필드)
{
  id: "xxx",
  title: "사전신청서 제출",
  dueDate: "2026-04-08",
  tripId: "yyy",   // ← NEW: 연결된 출장 ID (없으면 null)
}
```

#### 워크플로우 개선 진입점 3가지

| 진입점 | 설명 |
|--------|------|
| A. 출장 수정 폼 하단 | 저장 후 "관련 할 일 추가하기" 버튼 → 출장 정보 자동 채움 |
| B. 출장 카드(출장 페이지) | "✚ 할 일 추가" 버튼 → 출장 정보 자동 채움 후 수정 폼 열림 |
| C. TODO 수정 폼 내부 | "출장 연결(선택)" 드롭다운 → 활성 출장 목록에서 선택 |

#### 연계 삭제 정책
- 출장 삭제 시 연결 TODO는 삭제하지 않고 `tripId` → `null` 처리
- 사용자 팝업: "연결된 할 일 N건이 있습니다. 연결을 해제하고 출장을 삭제하겠습니까?"

## 리뷰 항목 4 — 대시보드에서 출장 상태 직접 변경

### 현황 분석
현재 **대시보드 > 요약 카드 > 출장 항목**을 클릭하면  
곧바로 **수정 폼**이 열립니다.  
출장 상태(계획됨 → 진행중 → 완료)를 변경하려면 반드시 **출장 페이지**로 이동해야 합니다.

대시보드 요약 카드에서 한 번에 상태를 바꿀 수 없어 **불필요한 페이지 이동이 발생**합니다.

### 해결 방향

#### A. 대시보드 출장 항목에 인라인 상태 버튼 추가
```
[수소연료전지] 세코 PDU 계약 미팅       D-1
📍 모아소프트(대전) · 2026.4.8 오전 9:00
                              [진행 시작 ▶]
```
- 출장 항목 우측 하단 또는 D-Day 배지 아래에 작은 상태 전환 버튼(▶ 진행 시작 / ✓ 완료) 추가
- 버튼 클릭 시 상태 즉시 변경 + 대시보드 리렌더링 (페이지 이동 없음)
- 영향 파일: `dashboard.js` `renderDashTrips()` 함수

## 리뷰 항목 5 — 대시보드 항목 클릭 UX 일관성 (상세 패널 도입)

### 현황 분석
현재 동작:
- **대시보드 > 출장 항목 클릭** → 즉시 수정 폼(Modal) 열림
- **대시보드 > 할 일 항목 클릭** → TODO 페이지 전체로 이동

반면 **출장 페이지 / TODO 페이지**에서는:
- 해당 수정 폼을 거치지 않으면 상세 내용을 볼 방법이 없음

이로 인해 **"보기"와 "수정"이 구분 없이 혼재**되어 사용자가 실수로 내용을 변경할 가능성이 있고,  
대시보드와 각 페이지의 UX 흐름이 달라 혼란이 발생합니다.

### 기획: 항목 클릭 시 "상세 패널" 먼저 표시

#### 클릭 흐름 변경

```
[기존]
대시보드 > 출장 항목 클릭 → 수정 폼 열림 (바로 편집)

[개선]
대시보드 > 출장 항목 클릭 → 상세 패널 열림 (읽기 전용)
                              └ [수정하기] 버튼 클릭 → 수정 폼 열림
                              └ [진행 시작] 버튼 → 상태 즉시 변경
                              └ [할 일 추가] 버튼 → TODO 수정 폼 열림 (출장 자동 연결)
                              └ [문서 작성] 버튼 → 문서 페이지 이동
```

#### 상세 패널 구성 (출장)
```
┌──────────────────────────────────┐
│ [수소연료전지] 세코 PDU 계약 미팅  ✕ │
│ 📍 모아소프트(대전)                │
│ 📅 2026.4.8 오전 9:00            │
│    ~ 2026.4.8 오후 5:00          │
│ 상태: [계획됨]  D-1               │
│ 메모: ...                        │
├──────────────────────────────────┤
│ 📋 연결된 할 일 (2건)              │
│   ☐ 사전신청서 작성  [높음]        │
│   ✓ 출장비 신청     [완료]         │
├──────────────────────────────────┤
│ [▶ 진행 시작]  [✚ 할 일 추가]     │
│ [📄 문서 작성]   [✏️ 수정하기]      │
└──────────────────────────────────┘
```

#### 상세 패널 구성 (할 일)
```
┌──────────────────────────────────┐
│ [ADD]RCS 측정 소형무인기 제안서    ✕ │
│ 📅 2026.4.3 마감  [높음]          │
│ 태그: 제안서, ADD                 │
│ 연결 출장: [수소연료전지] 세코 미팅  │
│ 메모: ...                        │
├──────────────────────────────────┤
│ [◐ 진행중으로 변경]  [✏️ 수정하기]  │
│ [✓ 완료 처리]                     │
└──────────────────────────────────┘
```

#### 기술 구현 방식
- 기존 수정 폼이 사용하는 `openModal()` 을 그대로 활용하되,  
  **읽기 전용 뷰 HTML** 을 전달하는 새 함수 `openDetailPanel(type, id)` 를 추가합니다.
- 수정 폼과 상세 패널은 동일한 모달 위에 표시되므로 추가 DOM 변경 최소화.

#### UX 일관성 원칙 (향후 가이드라인)
> **"대시보드에서 항목을 클릭하면 반드시 상세 패널이 먼저 열린다."**  
> **"수정 폼은 상세 패널 내의 [수정하기] 또는 각 페이지 내 수정 버튼을 통해서만 진입한다."**
